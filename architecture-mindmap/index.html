◈ Architecture Gaps & Action Items
Critical — Block Functionality (3 items)
No Emma AI inbound webhook handler
Inbound messages from Emma-connected channels (WhatsApp, IG, FB, Telegram) have no handler at /api/webhooks/emma. Without this, those channels are outbound-only. Emma team still building the payload + auth mechanism.
Emma Pending api/webhooks/emma/route.ts No route exists yet
RevRa AI Agent — no backend
The floating RevRaAIChat component (bottom-right on all user pages) has a UI but zero backend. lib/ai/tools.ts defines 13 LLM function-calling tools but there's no /api/ai/chat endpoint to wire them to. Needs to be built once Emma provides the right AI integration.
Future (Phase 5) app/api/ai/chat/route.ts Wire 13 tools from lib/ai/tools.ts
Active call flow — disconnected
/user/calls/active/page.tsx (live call UI with talking points, objection handlers, live notes) has no Twilio integration. No inbound call routing webhook, no active call connection. Future work once Emma handles Twilio.
Future app/user/calls/active/page.tsx Needs Twilio webhook + Emma connection
